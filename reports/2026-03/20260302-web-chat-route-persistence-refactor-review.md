## 🚀 优化建议（按优先级）

### P0 — 必须修复（不得合并）

无。

### P1 — 强烈建议（整改后合并）

1. **[问题]**：`/chat/invalid-id` 失败后只提示错误，不执行降级恢复，与报告验收不一致。  
   **[定位]**：`web/src/pages/Chat.vue:236-243`、`423-425`  
   **[建议]**：在 `handleSessionSwitch` 拉取历史失败时判断目标 ID 是否存在于 `sessions`，不存在则自动 `router.replace({ name: 'Chat', params: { id: sessions[0]?.id } })` 或 `createNewSession()`。

2. **[问题]**：新建会话时欢迎语可能被 watcher 触发的历史加载覆盖，造成 UX 回退。  
   **[定位]**：`web/src/pages/Chat.vue:297-310`、`403-409`、`237-239`  
   **[建议]**：
   - 方案 A：`createNewSession` 后先本地设置一个 `isNewSession` 标志，在 `handleSessionSwitch` 中遇到空历史时保留欢迎语。
   - 方案 B：把欢迎语生成迁移到 `handleSessionSwitch`，仅在“空历史且会话新建”条件下统一处理，避免竞争写入。

### P2 — 锦上添花（可遗留，记录到 tech-debt）

1. **[问题]**：报告写“全站纯中文注释”，但页面仍有中英混合注释，报告与代码描述不一致。  
   **[定位]**：`web/src/pages/Chat.vue:174`（及多处）  
   **[建议]**：同步修正文档描述，或统一注释规范并一次性清理。
