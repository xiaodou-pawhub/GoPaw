## 🚀 优化建议（按优先级）

### P0 — 必须修复（不得合并）

1. **[问题]** 流式通信协议回退：`useChatStream` 使用 `EventSource + GET query`，重新引入 URL 长度限制与内容暴露风险。  
   **[定位]** `web/src/pages/composables/useChatStream.ts:32-33`，`web/src/api/agent.ts:40-45`  
   **[建议]** composable 改为复用 `sendChatStream(sessionId, content, callbacks, { signal })`（POST）；保留 `AbortController` 中断能力，禁止走 `getChatStreamUrl`。

2. **[问题]** Chat 功能回退：文件上传入口与附件随消息发送逻辑被移除。  
   **[定位]** `web/src/pages/Chat.vue:102-126`（输入区已无上传按钮和待发送文件预览）  
   **[建议]** 将上传状态与发送拼装逻辑迁移到新 composable（如 `useChatComposer`），确保原有上传链路与附件发送能力完整保留。

### P1 — 强烈建议（整改后合并）

1. **[问题]** 删除按钮点击事件未阻止冒泡，可能触发父级 `selectSession` 导致误切会话。  
   **[定位]** `web/src/pages/Chat.vue:17`（列表项点击）+ `web/src/pages/Chat.vue:28`（删除按钮点击）  
   **[建议]** 在按钮点击上加 `.stop` 或在处理函数中显式 `event.stopPropagation()`。

### P2 — 锦上添花（可遗留，记录到 tech-debt）

   **[定位]** `web/src/pages/composables/useChatStream.ts:5`
