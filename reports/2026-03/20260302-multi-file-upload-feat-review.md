## 🚀 优化建议（按优先级）

### P0 — 必须修复（不得合并）
1. **[问题]**：文件内容直接拼接到 `content` 后通过 SSE GET URL 发送，存在 URL 长度硬上限，文件功能在实际场景不可用。  
   **[定位]**：`web/src/pages/Chat.vue:466`、`494`，`web/src/api/agent.ts:43`  
   **[建议]**：改为 POST 发送正文（含文本/附件引用），不要通过 query 传递大内容；若必须流式，先建立会话消息资源（服务端存储上传结果）再以短 ID 触发流式处理。  

### P1 — 强烈建议（整改后合并）
1. **[问题]**：图片只按扩展名识别，`DetectContentType` 结果未强约束，伪装文件可绕过图片分支校验。  
   **[定位]**：`internal/server/handlers/upload.go:85-96`  
   **[建议]**：对图片要求 MIME 前缀必须为 `image/`，否则拒绝；文本也建议限制可打印字符比例。  

2. **[问题]**：前端上传使用原生 `fetch`，未走统一 `api` 实例，缺少统一错误处理和拦截器行为。  
   **[定位]**：`web/src/pages/Chat.vue:423`  
   **[建议]**：抽到 `web/src/api/agent.ts`（如 `uploadFile(formData)`），统一请求链路。  

### P2 — 锦上添花（可遗留，记录到 tech-debt）
1. **[问题]**：上传大小与后缀白名单硬编码在 handler。  
   **[定位]**：`internal/server/handlers/upload.go:30-41`、`71`  
   **[建议]**：迁移到配置项并在启动时校验。
