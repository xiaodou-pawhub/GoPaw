## 🚀 优化建议（按优先级）

### P1 — 强烈建议（整改后合并）

1. **[问题]**：删除当前会话时未关闭 SSE，可能出现“已删除会话仍写入消息”现象。  
   **[定位]**：`web/src/pages/Chat.vue:221-227,353-393`  
   **[建议]**：维护 `currentEventSource` 并在删除当前会话前执行 `close()` + 状态重置。

2. **[问题]**：确认弹窗文案硬编码，缺少 i18n。  
   **[定位]**：`web/src/pages/Chat.vue:216`  
   **[建议]**：新增 `chat.deleteConfirm` 等国际化 key。

3. **[问题]**：后端删除接口未显式校验空 id。  
   **[定位]**：`internal/server/handlers/agent.go:137-146`  
   **[建议]**：增加 `if sessionID == "" { return 400 }` 防御分支。

### P2 — 锦上添花（可遗留，记录到 tech-debt）

1. **[问题]**：删除后会话重选逻辑分散在 `loadSessions` 和 `handleDeleteSession`。  
   **[建议]**：抽象 `syncSessionAfterDelete(deletedId)`，减少状态分支重复。
