## 🚀 优化建议（按优先级）

### P0 — 必须修复（不得合并）

无。

### P1 — 强烈建议（整改后合并）

1. **[问题]**：执行记录创建失败后仍继续更新，且 `UpdateRun` 错误被忽略，可能导致历史记录丢失且不可见。  
   **[定位]**：`internal/scheduler/manager.go:255-259`、`277`、`290`、`294`  
   **[建议]**：
   - 增加 `recordEnabled := (err == nil && runID != "")` 控制，仅在创建成功后调用 `UpdateRun`。
   - 对 `UpdateRun` 返回值做日志告警（至少 `Warn`），避免静默失败。

### P2 — 锦上添花（可遗留，记录到 tech-debt）

1. **[问题]**：`ListRuns` 的 `limit` 解析使用 `fmt.Sscanf`，非法输入不会显式报错。  
   **[定位]**：`internal/server/handlers/cron.go:126-128`  
   **[建议]**：改用 `strconv.Atoi` 并对非法值返回 `400`，提升 API 可预期性。
