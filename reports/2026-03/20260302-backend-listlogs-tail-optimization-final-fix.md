### P1-3: 文件末尾带换行时混入额外空日志

**问题**: `strings.Split` 在文件末尾有换行符时会产生额外空元素。

**定位**: `logs.go:91-101`

**修复前**:
```go
if line == "" && i == len(lines)-1 && cursor > 0 {
    // 块末尾的空行是因为 split 产生的，跳过
    continue
}
```

**修复后**:
```go
// 修复 P1-3: 文件末尾换行产生的额外空行
// 如果是第一个块（cursor+readSize==filesize）且是最后一行且为空，跳过
isFileEnd := (cursor + readSize == filesize)
if line == "" && i == len(lines)-1 && isFileEnd {
    continue
}
// 块中间的空行是真实的，保留
```

**效果**: 
- ✅ 文件末尾换行不会产生额外空行
- ✅ 真实空日志行会被保留
- ✅ 块中间的空行正确保留

### P1-4: 返回顺序从"旧→新"变成"新→旧"

**问题**: 反向读取导致返回顺序反转，存在行为兼容风险。

**定位**: `logs.go:129-133`

**修复前**:
```go
// 从后向前读，返回 [新...旧] ❌ 与原实现不兼容
```

**修复后**:
```go
// 修复 P1-4: 反转结果数组，保持"旧→新"顺序（与原实现兼容）
// 原实现是顺序扫描，返回 [旧...新]
// 本实现是从后向前读，需要反转为 [旧...新]
for i, j := 0, len(result)-1; i < j; i, j = i+1, j-1 {
    result[i], result[j] = result[j], result[i]
}
```

**效果**: 
- ✅ 返回顺序与原实现一致 [旧...新]
- ✅ 前端 UI 显示正确（旧日志在上，新日志在下）
- ✅ 行为兼容，无回归风险
