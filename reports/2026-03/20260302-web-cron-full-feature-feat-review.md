## 🚀 优化建议（按优先级）

### P0 — 必须修复（不得合并）
无。

### P1 — 强烈建议（整改后合并）

1. **[问题]** “活跃时间窗口（可选）”与实际行为不一致。新建任务默认携带 `09:00`/`18:00`，导致任务被隐式限时执行。  
   **[定位]** `web/src/pages/Cron.vue:79`、`185-194`、`286-295`；后端执行窗口：`internal/scheduler/manager.go:319-330`  
   **[建议]** 将默认值改为空字符串 `''`，仅用户显式选择时间时再提交；或增加“启用时间窗口”开关后再传 `active_from/active_until`。

2. **[问题]** 表单校验未在提交时强制执行，用户可在未触发表单事件时直接提交。  
   **[定位]** `web/src/pages/Cron.vue:300-307`  
   **[建议]** 增加 `const formRef = ref<FormInst | null>(null)` 并在 `handleSubmit` 开头执行 `await formRef.value?.validate()`；失败直接 return，不请求后端。

### P2 — 锦上添花（可遗留，记录到 tech-debt）

1. **[问题]** 报告宣称“彻底清理 any”，但 API 层仍存在 `any`。  
   **[定位]** `web/src/api/cron.ts:6,32`  
   **[建议]** 定义明确响应类型：`{ jobs: CronJob[] }`、`{ runs: CronRun[] }`，移除 `any`。

2. **[问题]** Cron 页面存在较多硬编码文案，i18n 一致性不足。  
   **[定位]** `web/src/pages/Cron.vue:7,44,58,67,79,111,132,136,320...`  
   **[建议]** 抽到 `web/src/locales/index.ts`，统一用 `t('...')`。
